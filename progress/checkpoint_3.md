## 技术方案

### 分辨率策略

初始方案为将图片长边缩至 512px 后调用 API，测试后发现标注密集的驾驶舱界面图在 512px 下文字模糊，导致模型无法辨认标签，产生系统性幻觉。

**最终决策**：使用原图直接调用，不做缩放。代价是单张 token 消耗从约 8600 升至约 14000-37000（取决于图片尺寸），总费用从预估 $0.05 升至实际 $3.10，在可接受范围内。

### Prompt 设计

经过三轮迭代测试（共 32 张样本图片）确定最终 Prompt：

**System Prompt**：
```
你是一个飞行模拟器界面分析助手。请严格只描述图片中实际可见的内容，
禁止推断或补充图中未显示的信息。用中文回答。
```

**User Prompt**：
```
用一句中文描述这张图片的主体内容，重点保留图中出现的关键英文标签和中文标注文字
（最多列举5个最重要的），控制在100字以内。只描述图中可见内容，不推断图中未显示的信息。
如果图中文字模糊无法确认，只描述可清晰辨认的内容，不猜测或编造标签名称。
如果图中没有可辨认的文字标注，用简短的位置或类型描述代替，例如：座舱内部右侧面板特写、导弹实物侧视图。

输出格式要求：直接描述内容，不要以"这张图片展示了"、"图片中显示"等开头。
示例输出：驾驶舱音频控制面板，标注有INTERCOM、TACAN、ILS（OFF状态）、HOT MIC开关。
```

关键设计决策：
- **自然语言优于固定格式模板**：手册图片类型多样（驾驶舱面板、示意图、实物照片、飞行程序图等），强制三段式格式在无标注图片上效果差，改为自然语言指令后适应性更好
- **退路机制**：对无可辨认文字的图片提供"位置/类型描述"退路，防止模型因无标签可列而编造内容
- **few-shot 示例**：提供一条示例输出，消除"这张图片展示了"等冗余开头

### 摘要注入格式

摘要以固定前缀追加到 Chunk text 末尾，每张图片一行：

```
原始手册文字内容...
[IMG_CAPTION] 驾驶舱音频控制面板，标注有INTERCOM、TACAN、ILS（OFF状态）、HOT MIC开关。
[IMG_CAPTION] 航向指示器，标注有距离指示器052NM、航线设置窗口、HDG旋钮。
```

`[IMG_CAPTION]` 前缀便于后期 ablation 测试（可一键去除摘要内容对比检索效果差异）。

### 批量处理与容错

- **断点续传**：每张图片处理完成后立即写入 `captions_cache.json`，中途崩溃不丢失已完成结果
- **图片去重**：以图片路径去重后再调用 API，边界双写图片不重复计费
- **失败重试**：单张图片失败后指数退避重试 3 次（退避基数 5 秒），三次失败记录为 `CAPTION_FAILED`
- **重跑支持**：`CAPTION_FAILED` 条目在重跑时被视为未缓存，自动重新处理

### Rate Limit 处理

全量运行过程中触发了 OpenAI API Rate Limit，775 张图片首轮失败。应对策略：
1. 并发数从 5 降至 2
2. 重试退避基数从 1 秒升至 5 秒
3. 利用断点续传机制分三轮完成全量处理，已成功的图片不重复调用

## 主要困难与解决过程

### 1. 512px 分辨率导致系统性幻觉

**问题**：第一轮测试（512px）中，多张驾驶舱界面图的摘要出现严重幻觉——模型编造了图中不存在的英文标签（如 `Throttle Control`、`Pressure Gauge`），或将通用控件名当作实际标注输出。根本原因是缩放后标注文字模糊，模型看不清时倾向于"合理猜测"。

**解决**：改用原图直接调用，同时在 Prompt 中加入明确的防幻觉指令和退路机制。第二轮测试幻觉问题大幅改善，主要错误从"编造标签"降为"偶发小字误读"（如 VHF 读为 UHF），属可接受范围。

### 2. Rate Limit 触发导致大量失败

**问题**：首轮全量运行时并发数设为 5，连续请求触发 OpenAI Rate Limit，775/1068 张图片失败（失败率 72%）。

**解决**：降低并发数至 2，加长重试退避时间，利用断点续传分轮处理，最终 CAPTION_FAILED 残留归零。代价是总运行时间增加，分三轮完成。

### 3. 无文字标注图片的输出不稳定

**问题**：部分图片（如驾驶舱全景图、历史照片）没有明显的文字标注，模型不知道该输出什么，在 512px 阶段会编造通用标签填充，在原图阶段则可能输出过于泛泛的描述。

**解决**：Prompt 中明确提供退路："如果图中没有可辨认的文字标注，用简短的位置或类型描述代替"，并给出具体示例（座舱内部右侧面板特写）。测试验证该退路有效激活，如 `page_102_img_0`（dcsjf17）输出"座舱内部右侧面板特写，标注有红色指针的仪表、连接杆、气管"，未产生幻觉。

## 已知局限（不影响 MVP）

- 极少数图片中的小字英文标签仍存在误读（如 VHF 读为 UHF），属模型视觉精度限制，非系统性问题
- 边缘截断的正文文字（排版溢入图片区域的段落文字）偶有被当成标注内容读取（如 dcsfa18c chunk_0022 的导弹图）；Prompt 中"主体内容"的指引有所缓解但未完全消除
- 超大 Chunk 切分后的子 Chunk（如 `dcsjf17_0003_part*`）继承了父 Chunk 的全部图片，部分子 Chunk 的摘要极长（21 张图的摘要拼接）；这是阶段二切分策略的已知局限，不在本阶段范围内

## 阶段三设计决策（相对原技术方案的调整）

| 原计划 | 实际决策 | 原因 |
|--------|---------|------|
| 图片缩至 512px 调用 | 使用原图直接调用 | 512px 下标注文字模糊导致系统性幻觉，原图质量显著更好 |
| 预估费用 $0.05 | 实际费用 $3.10 | 原图 token 消耗约为 512px 的 2-4 倍；总费用仍在可接受范围 |
| 固定三段式输出格式 | 自然语言指令 + few-shot 示例 | 图片类型多样，固定格式对无标注图片效果差；自然语言指令适应性更强 |
| 并发数 5-10 | 最终并发数 2 | 高并发触发 Rate Limit；降低并发后稳定性显著提升 |
